# Audio Full-Duplex 常见问题

## 什么是 Audio Full-Duplex 模式？

Audio Full-Duplex（音频全双工）模式支持**纯音频的双向实时对话**。你和模型可以同时说话，无阻塞。类似于真实的电话通话体验，但不包含视频。

> 注意：此功能目前为实验性质，模型在说话时可能无法立即响应新的提问。建议使用耳机以获得最佳体验。下次模型版本会优化这个问题。

## 回复过于简短？

由于模型训练语料有局限性，回复可能较为简洁。**可以通过左侧栏调高 Length Penalty 参数，达到 1.3 实现更长的回复长度和更好的共情能力。**默认值为 1.05 回复较为简洁。但已知问题是在 Length Penalty = 1.3 时，可能难以语音打断。下个模型版本会重点提升。

## Preset System Prompt 会影响音色、韵律吗？

会。当前这组 preset 和 system prompt 文本仍然会影响回复语言、说话风格、节奏感和韵律表现。当前部署默认使用预设/默认参考音频，但没有开放单独替换 TTS 参考音频的入口。若要真正替换 C++ 侧的 TTS 合成音色，需要参照[换音色指南](https://github.com/OpenSQZ/MiniCPM-V-CookBook/blob/main/deployment/llama.cpp-omni/%E6%8D%A2%E9%9F%B3%E8%89%B2%E6%8C%87%E5%8D%97.md)自行替换 `prompt_cache.gguf` 和相关资源。

## 可以自定义系统提示词吗？

可以，通过 Preset System Prompt 点击 Advanced 按钮修改 system prompt 文本。修改后会影响模型的角色、行为、回复风格，以及语音回复的表达方式；但不会提供单独替换 TTS 参考音频的入口。


## 和 Half-Duplex 有什么区别？

| 特性 | Half-Duplex | Audio Full-Duplex |
|------|------------|-------------------|
| 通话方式 | 轮流对话 | 可同时说话 |
| 打断 | 需等模型说完 | 可随时打断 |
| 延迟 | 较低 | 稍高 |
| 稳定性 | 更稳定 | 实验性 |

## Live 模式和 File 模式有什么区别？

- **Live**：使用麦克风进行实时语音输入
- **File**：上传音频文件作为输入，可选择仅使用文件音频或混合麦克风

## 为什么建议使用耳机？

全双工模式下，模型的回复声音可能通过扬声器传到麦克风中，造成回声。使用耳机可以有效避免这个问题，让模型更准确地听到你的声音。

## Delay 该设多少？

- **默认 200ms**：适合大多数场景
- **降到 50-100ms**：低延迟但可能有音频断续
- **调高到 300-500ms**：更稳定，适合网络较差的情况

## Force Listen 按钮是做什么的？

点击 **Force Listen** 可以强制模型进入监听状态，中断它当前的回复。当你想打断模型说话时使用。

## Pause 按钮有什么效果？

暂停当前会话。暂停期间不发送音频数据，恢复后可以继续对话。适合临时中断的场景。
